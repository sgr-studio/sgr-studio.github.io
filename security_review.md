# sgr-studio.github.io セキュリティレビューレポート

## 1. はじめに

この度は、貴社ウェブサイト `sgr-studio.github.io` がGoogle Safe Browsingによって「危険なサイト」として判定された件について、ご依頼いただきありがとうございます。本レポートでは、リポジトリのソースコードを詳細に調査した結果、警告の原因と考えられる潜在的なリスク要因を特定し、具体的な修正案を提示します。

## 2. 調査概要

調査は以下の手順で実施しました。

1.  **リポジトリのクローン**: `sgr-studio/sgr-studio.github.io` リポジトリの全ファイルをローカル環境に複製しました。
2.  **静的コード解析**: HTML、CSS、JavaScriptファイルの内容を精査し、以下の観点から危険と判断されうる要素を探索しました。
    *   不審な外部リンクやスクリプトの読み込み
    *   個人情報（パスワード、クレジットカード情報など）の入力を促すフォーム
    *   マルウェアや不要なソフトウェアのダウンロードを誘導する可能性のある記述
    *   難読化された、あるいは意図が不明瞭なコード

## 3. 検出された潜在的リスク要因と分析

詳細な調査の結果、直接的なマルウェアやフィッシング詐欺を意図した悪質なコードは発見されませんでした。しかし、Googleの自動クローラーが「欺瞞的なサイト」と誤判定する可能性のある、いくつかの要因が特定されました。これらの要因が複合的に影響し、今回の警告につながったものと推測されます。

以下に、具体的なリスク要因と分析結果を詳述します。

### 3.1. 外部ドメインからのリソース読み込み

複数のHTMLファイルにおいて、外部ドメインからCSSフレームワークやアイコンライブラリを読み込んでいます。これ自体は一般的な手法ですが、読み込み先のドメインの信頼性が低い場合や、過去にセキュリティインシデントがあった場合、リスクと見なされる可能性があります。

| ファイル | 行番号 | 読み込み先URL | 目的 | 潜在的リスク |
| :--- | :--- | :--- | :--- | :--- |
| `apps.html` | 12 | `https://unpkg.com/boxicons@2.1.4/css/boxicons.min.css` | アイコン表示 | CDNの信頼性、バージョンの脆弱性 |
| `contact.html` | 13 | `https://unpkg.com/boxicons@2.1.4/css/boxicons.min.css` | アイコン表示 | CDNの信頼性、バージョンの脆弱性 |
| `index.html` | 12 | `https://unpkg.com/boxicons@2.1.4/css/boxicons.min.css` | アイコン表示 | CDNの信頼性、バージョンの脆弱性 |
| `news.html` | 12 | `https://unpkg.com/boxicons@2.1.4/css/boxicons.min.css` | アイコン表示 | CDNの信頼性、バージョンの脆弱性 |
| 全てのHTML | - | `https://sgr-studio.github.io/database/data/icon/favicon.png` | ファビコン表示 | 異なるリポジトリからのリソース読み込み |

特に、`unpkg.com` は誰でもnpmパッケージを公開できるCDNであり、利便性が高い一方で、悪意のあるパッケージがホストされる可能性もゼロではありません。また、ファビコンが別リポジトリのURLで指定されている点も、クローラーに不整合と判断される可能性があります。

### 3.2. Googleフォームの埋め込み

`contact.html` にて、Googleフォームが `<iframe>` タグを用いて埋め込まれています。これは一般的なお問い合わせフォームの実装方法ですが、Google Safe Browsingは `<iframe>` による外部コンテンツの埋め込み、特にフォームの埋め込みに対して慎重な評価を行う傾向があります。

> 攻撃者がユーザーを騙して個人情報を入力させるために、正規のサイトに見せかけたページに外部のフォームを埋め込む手口が存在します。

貴サイトのフォームは正当な目的で使用されていますが、自動スキャンではその意図まで汲み取ることが難しく、「個人情報を収集する可能性がある」と機械的に判断された可能性があります。

| ファイル | 行番号 | 埋め込みURL | 目的 |
| :--- | :--- | :--- | :--- |
| `contact.html` | 76 | `https://docs.google.com/forms/.../viewform?embedded=true` | お問い合わせフォーム |

### 3.3. ユーザーの操作をトリガーとする動的処理

複数のJavaScriptファイルに、ユーザーの操作（マウスクリック、ドラッグなど）に応じて動的な処理を行うコードが含まれています。これらのコードはサイトのインタラクティブ性を高めるためのものですが、一部の処理が「ユーザーを欺くための挙動」と誤解される可能性があります。

*   **`pickup.js`**: `ondblclick` 属性で新しいウィンドウを開く (`window.open`) 処理があります。ポップアップウィンドウは、しばしば広告やフィッシングサイトへの誘導に利用されるため、警告の対象となりやすいです。
*   **`snow.js`**: 特定の日付（12月22日）になると、`alert()` を表示し、雪を降らせるアニメーションを開始する処理があります。`alert()` はデバッグ目的でよく使われますが、ユーザーを驚かせたり、システム警告を装ったりするためにも利用できるため、これもリスク要因と見なされることがあります。

| ファイル | 行番号 | コード | 潜在的リスク |
| :--- | :--- | :--- | :--- |
| `pickup.js` | 28 | `ondblclick="window.open(...)"` | ポップアップウィンドウによる外部サイトへの遷移 |
| `snow.js` | 92 | `alert(...)` | 予期しないアラート表示 |

### 3.4. 複数のプロジェクトへのリンク

`pickup.json` には、`sgr-studio.github.io` ドメイン以外の複数のGitHub Pagesサイトへのリンクが含まれています。これらのリンク先サイトのコンテンツや信頼性も、Googleの評価に影響を与える可能性があります。もしリンク先のいずれかがセキュリティ上の問題を抱えていた場合、リンク元のサイトの評価も低下することがあります。

| ファイル | 行番号 | リンク先URL |
| :--- | :--- | :--- |
| `pickup.json` | 26 | `https://suchgamer.github.io/2shinsuu/` |

## 4. 修正提案

以上の分析に基づき、Google Safe Browsingの警告を解除し、将来的なリスクを低減するための具体的な修正案を以下に提案します。

### 4.1. 外部リソースのローカル化

外部ドメインへの依存を減らし、セキュリティを向上させるために、CSSやアイコンなどのリソースをリポジトリ内にダウンロードし、ローカルパスで読み込むことを強く推奨します。

1.  **Boxicons**: `boxicons.min.css` ファイルと関連するフォントファイル（woff, woff2など）をダウンロードし、`css` ディレクトリや新しく作成する `fonts` ディレクトリに配置します。その後、HTMLファイル内の `<link>` タグの `href` 属性をローカルパスに変更します。
2.  **Favicon**: `favicon.png` を `sgr-studio.github.io` リポジトリ内の `media/icon` ディレクトリなどに配置し、HTMLファイル内のパスを相対パス（例: `media/icon/favicon.png`）に変更します。

### 4.2. お問い合わせフォームの実装変更

`<iframe>` によるフォームの埋め込みは避け、より安全な代替案を検討します。

*   **代替案1（推奨）**: Googleフォームへの直接リンクに変更する。`contact.html` にはフォームの埋め込みをやめ、「お問い合わせはこちら」のようなボタンやリンクを設置し、クリックするとGoogleフォームのページが新しいタブで開くようにします。これにより、ユーザーは公式のGoogleドメインで情報を入力していることを確認でき、安心感が高まります。
*   **代替案2**: 静的サイト向けのフォームバックエンドサービス（例: Formspree, Netlify Formsなど）を利用する。これにより、サイトのデザインに合わせた独自のフォームを維持しつつ、安全にデータを受け取ることが可能になります。

### 4.3. 動的処理の修正

ユーザーに誤解を与える可能性のあるJavaScriptの処理を修正します。

1.  **`pickup.js` の `window.open`**: `ondblclick` ではなく、通常のシングルクリック (`onclick`) でリンク先に遷移するように変更します。また、`target="_blank"` を `<a>` タグに設定することで、新しいタブで開く動作は維持できます。ダブルクリックという直感的でない操作と `window.open` の組み合わせを避けることで、リスクを低減できます。
2.  **`snow.js` の `alert`**: ユーザーへの通知が必須でない限り、`alert()` は削除することを推奨します。もし何らかのメッセージを伝えたい場合は、ページ上部にバナーを表示するなど、よりユーザーフレンドリーな方法を検討してください。

### 4.4. リンク先の監査

`pickup.json` からリンクされている外部サイトの内容を再確認し、現在メンテナンスされていない、あるいはセキュリティ上の懸念があるプロジェクトへのリンクは削除または更新することを検討してください。

## 5. まとめと次のステップ

本調査により、貴社サイトには悪意のあるコードは存在しないものの、Google Safe Browsingの自動クローラーによって「欺瞞的」と誤判定されうる複数の要因が存在することが明らかになりました。上記で提案した修正を実施することで、これらのリスクは大幅に軽減され、Googleからの評価が改善される可能性が高いと考えられます。

**推奨される次のステップ:**

1.  本レポートに記載された修正案を適用します。
2.  修正後、Google Search Consoleにログインし、セキュリティの問題が検出されていないか確認します。
3.  もし問題が引き続き表示されている場合は、Google Search Consoleを通じて再審査をリクエストします。

ご不明な点がございましたら、お気軽にお問い合わせください。
